整体技术架构

基于 uni-app + uView Plus 的分层架构设计

📦 技术选型汇总

框架uni-app (Vue 3 + TypeScript)
UI 库uView Plus 3.x
状态管理Pinia
HTTP 请求uni.request + 请求拦截器
设计风格蓝色专业风 (#4A90D9)
数据模式Mock 数据 + API 接口预留
账号体系多角色合一，支持身份切换

🏗️ 分层架构

页面层 (Pages)
登录 · 主页 · 消息 · 功能 · 我的
↓
组件层 (Components)
请假表单 · 访校二维码 · 排班日历 · 消息卡片 · 通用组件
↓
状态层 (Stores - Pinia)
userStore · messageStore · featureStore
↓
服务层 (Services / API)
mock 数据 + 真实 API 接口预留
↓
工具层 (Utils)
请求封装 · 权限判断 · 日期处理 · 常量定义
数据流：页面 → 组件 → Store → API Service → Mock/真实后端
权限流：用户登录 → 角色识别 → Store 存储 → 页面/组件按权限渲染
